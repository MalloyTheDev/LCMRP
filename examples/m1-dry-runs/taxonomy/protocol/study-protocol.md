# SYNTHETIC-DRY-RUN taxonomy study protocol

Applicable layer: Layer 1 — Foundational Research  
Artifact status: FROZEN synthetic contract fixture  
Evidence status: NON-EVIDENTIARY  
Mechanism evidence labels: Not applicable

This protocol exists only to exercise the `STRUCTURAL_OR_TAXONOMY_EVALUATION` serialization path. It does not register a real study, assert a result, endorse either candidate organization, or select any implementation component.

## Research question

Can the contract encode different identity, cardinality, context, and classification commitments for the kind-first and role-first organizations across fixed constructed cases?

## Falsifiable fixture claim

The frozen rules and case fields are sufficient to serialize an organization-specific disposition for every case. The fixture claim is rejected if a required field is absent, the rules collapse into indistinguishable outputs on every boundary case, or the all-case ledger is incomplete.

## Method and comparisons

The unit is a constructed case paired with one organization. Organization K and Organization R are processed independently using only their frozen rules. Positive cases exercise straightforward encodings; negative cases exercise declared constraint failures; boundary cases expose the organizations' intentionally incompatible commitments.

No human judgment, participant data, stochastic process, external corpus, model inference, or product scenario is used. The dry run does not test whether the constructed cases are representative.

## Planned analyses

1. `ANALYSIS-SYNTHETIC-ORG-COMPARISON` deterministically serializes the two organization commitments across the frozen cases.
2. `ANALYSIS-SYNTHETIC-BOUNDARY-ADJUDICATION` would test independent adjudicator agreement, but is intentionally `NOT_RUN` because this fixture uses no human or independent evaluator.

Each planned analysis must receive exactly one published atomic record, including an explicit terminal disposition when no observation exists.

## Security, privacy, and retention

Raw-byte digests protect fixture identity against accidental substitution. All content is artificial and contains no human subjects or participant data. The complete fixture family, including mixed, inconclusive, and not-run dispositions, is retained together. Digest integrity does not establish authorship, authenticity, scientific validity, or independent review.

## Rejection and stop rules

- Reject the fixture claim if any frozen case lacks an organization-specific serialized disposition.
- Invalidate an analysis if exact subject, profile, source, protocol, or configuration bindings fail.
- Halt rather than infer a missing context or classification.
- Preserve every failure or non-execution outcome.

## Limitations

The cases are authored to exercise the declared differences, so successful processing is tautological contract coverage rather than empirical support. No external validity, biological validity, usefulness, security effectiveness, novelty, adoption, validation, or mechanism maturity conclusion can follow.
